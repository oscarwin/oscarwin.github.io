## 为什么写这一篇文章

我是从校招开始接触到牛客网的，和很多人一样在牛客网刷题，在牛客网看面经，不得不说这对于我校招给予了很大的帮助。和很多人一样，也是半路出家，学的专业是机械工程，在学校期间接触过一点简单的 c++ 编程，阴差阳错跳进了互联网的坑里。

为了能在毕业年的秋招拿到互联网大厂的 offer，我开始筹划怎样补足计算机的知识和面试技能？那时的我计算机专业的课程我一门没有上过，程序员面试考什么我一点不知道。我不怕努力，但是我怕找不到努力的方向。这也是我写这篇文章的原因，为那些和我曾今一样不知道学习路线的人给予一点个人的经验。我校招选择了京东，如今在字节跳动，从开始认真写代码到如今已经有三年光阴。说这个是想告诉那些还在犹豫不决的人，只要愿意付出，零基础也不是问题。

## 岗位选择

刚开始确定了我要成为一名互联网程序员时，我除了会一点 c++ 以外，其他相关知识为零。更要命的是，我不知道该按照什么路线来学习，身边也没有可以给我指引的同学，认识的师兄都是找的机械行业的工作。当然，懂得使用搜索引擎的人就没有解决不了的问题，我开始搜相关的信息，然后梳理成章来到了牛客网，那时候的牛客网还没有如今知名度。我看了很多牛客网的面经和知乎的回答，大概知道了面试官要考察什么了。但是在开始准备之前，我还需要做一个决定，我到底要选择什么岗位，因为是网络上零碎信息的拼凑，我对前端后端工作的了解并不那么清楚，当时就只知道前端是写界面的，后端是写服务器的，具体使用什么样的技术栈，发展上是什么区别我并不清楚，反正当时觉得我会的 c++ 和后端可能还有一点关系，就选择成为了一名后端工程师。

如果你也有这样的困惑，而且时间还充足的话，你可以自己写一个非常简单的博客系统，能发文章和看文章就行。后端你可以选择 python 语言或者 go 语言，这两门语言的上手成本很低，前端就需要了解 HTML、CSS、JavaScript。这些你都能从网上找到教程，不过可能要费很大一番功夫，因为每遇到一个知识点你就要去了解，安装数据库，配置环境可能对你都是一些挑战，特别是没有人指导的话。如果你能弄出来那么恭喜你对一个完成的互联网产品背后的技术工作就有了了解。你可以根据你自己的兴趣方向选择是前端还是后端，当然如果你是大数据、算法岗等就另当别论了。

当然你可能还没有任何基础，上面的工作对你而言还有些困难，没关系你可以先从学习一门语言开始。我是因为大学参加过一些比赛，所以有一点 c 语言基础就学习了 c++，但是这门语言确实很难掌握。如果没有任何其他语言的基础还是建议你先学习 python，这个语言上手简单，功能强大，可以写服务器、可以写爬虫。后面刷算法题也可以用 python 来写。

如果你没有那么时间了，只是想先找一份工作，进入到这个行业来，那么你只要先学会一门语言就可以继续往下看了。Python、Go、C/C++、Java 都可以。

## 理论学习
有了目标可以开始准备了，面试考察最多的就是操作系统原理、网络、数据结构与算法。毫不意外，以上知识点我都不懂。于是我买来了[《数据结构与算法》](https://book.douban.com/subject/1139426/)、[《TCP/IP详解、卷一》](https://book.douban.com/subject/1088054/)、[《UNIX高级环境编程》](https://book.douban.com/subject/25900403/)、[《现代操作系统》](https://book.douban.com/subject/3852290/)开始自学计算机的基础理论知识的漫长道路。

### 网络
这几本书本本都是圣经一样的存在，困难是可想而知的，我清晰的记得《TCP/IP详解卷一》这本书网上评分很高，但是我看的很痛苦。且不说看的过程有多枯燥，即使咬牙看完了一遍，对计算机网络仍然一脸懵逼，我很沮丧。于是去找更易懂的教材，我又找到了一本叫[《图解TCP/IP》](https://book.douban.com/subject/24737674/)，顿时感觉打通了网络的任督二脉，几天就把这本书看完了，对网络也有了整体的了解，但是易懂的书往往讲的也比较浅。然后我再去重读《TCP/IP详解卷一》我才发现这确实是一本好书，不过不适合作为网络的入门书籍。所以当你要去探索一个未知的领域时，并不是最权威的书籍就是最适合你的，至少可能一开始不是。《TCP/IP详解卷一》这本书我看了不下三遍，不过那时候主要也只是看，没有什么实践，也不知道如何实践。但是至少面试没啥问题了，以致于我去面美团的时候，面试官让我讲一下 TCP 和 UDP，我滔滔不绝说了十多分钟，然后面试官不得不打断我说：好了，好了，看来你网络掌握的不错。

### 操作系统
操作系统也是个难啃的骨头，不过《现代操作系统》对于新手是友好的，特别是关于操作系统调度、虚拟内存和文件存储需要反复看，每次看都有新的认识。后来还看了一下《Linux内核设计与实现》，这本书很薄，作者用比较简短的篇幅就帮助我们窥探了Linux 内核的全貌，很推荐。腾讯在我印象里是比较喜欢问操作系统的，当时很想去腾讯，所以操作系统看得比较多。关于内存淘汰的 LRU 算法是需要能够手写出来的。

### 数据结构与算法
前面介绍的操作系统和网络，主要是理论学习，基本没有太多需要写代码的场景。学习《数据结构与算法》和《UNIX高级环境编程》时就可以锻炼编码的能力了。学习数据结构与算法最简单最有效的方法就是把所有的数据结构和算法自己手写一遍，比较复杂的红黑树、跳跃表这种可以不撸，但是原理要懂。数据结构与算法搞定了就可以开始刷题了，先刷完《剑指offer》，再刷LeetCode，罗马不是一天建成的，刷题也是一个需要长期坚持的事情。需要强调一下的是，很多同学在刷 LeetCode 的题时按照题号或者难易程度去刷的，这都不建议。**正确的刷题姿势应该是按照类型（栈、树、链表、字符串等）去刷，每个类型刷上十多道题就对这种类型题的解题思路有了系统的认知**。撸完了《数据结构与算法》有性趣，啊呸，有兴趣可以看《算法》和《算法大全》，都是很好的书，算法题不只是在校招时有用，在你整个职业生涯中都将发挥无比巨大的作用。

### Linux 编程
《UNIX高级环境编程》这本书是将 Linux 系统编程的，讲 Linux 的各种系统调用，讲各种 IO 模型，因为是服务端开发，所以肯定需要对 Linux 有足够的了解。这本书里的代码事例都是 c 语言写的，因为 Linux 对外提供的系统调用都是 c 接口。这本书事无巨细把 Linux 系统编程的每个细节都谈到了，但是也加大了学习的成本，如果可以重来，我会对书店老板说：给我来一本[《Linux系统编程手册》](https://book.douban.com/subject/25809330/)。《Linux 系统编程手册》这本书内容与《UNIX高级环境编程》是一样的，但是学起来更容易。Linux 系统编程的学习目标很明确，把各种echo服务器模型自己写一遍，就可以把大部分知识都贯通了，我这里的 echo 服务器模型是指：**单进程单线程echo服务器**，**多进程echo服务器**，**单进程多线程echo服务器**，**单进程IO多路复用echo服务器**，**多进程IO多路复用echo服务器**，**多线程IO多路复用echo服务器**。你可能听说过 apache、nginx、muduo、redis、libevent等，这些大名鼎鼎的网络库或者web服务器，底层的网络模型都逃不出上面所列举的范围。弄懂上面这些模型的演变过程，以及他们的优劣，你肯定就会了解到什么是 C10K 问题？你会被阻塞IO、非阻塞IO、同步IO、异步IO、IO多路复用搞得头都大了，别放弃，搞清楚这些你就超越大多数后端工程师。搞清楚这些后面看Redis源码，看libevent源码都是水到渠成的事。当然你可能不会 c/c++，你可以选择学习 c++，也可以用 Java 或者 go 来实现上线的服务器模型。为什么没有 Python 了，因为曾经用 Python 的字节跳动、知乎、哔哩哔哩都重构为 go 了，对于复杂业务的大型分布式系统 Python 搞不定了。

### 语言
当然校招生一般会写上自己会什么语言，面试官如果对你掌握的语言比较熟悉，那么肯定也会问你语言相关的问题。如果你最擅长的是 c++，那么要看[《C++ primer》](https://book.douban.com/subject/25708312/)、[《Effective C++》](https://book.douban.com/subject/5387403/)、[《深度探索C++对象模型》](https://book.douban.com/subject/10427315/)。如果你最擅长的是 go，那么先看[官方文档](https://golang.org/doc/)，如果觉得英文文档有压力，可以去这里找中文书籍看一下，[https://github.com/unknwon/go-study-index](https://github.com/unknwon/go-study-index)。最后可以看这个系列的[博客](https://draveness.me/golang/)，深入源码来分析 go 语言。至于 Java 我不熟悉就不瞎逼逼了。

### HTTP
作为互联网使用最广泛的 HTTP 协议怎么能够不考呢？HTTP 协议可以看 《图解 HTTP 协议》以及 MDN 的文档[https://developer.mozilla.org/en-US/docs/Web/HTTP](https://developer.mozilla.org/en-US/docs/Web/HTTP)。对于 HTTP 协议中涉及的安全和认证问题要特别关注。安全问题包括 CSRF 和 XSS。认证问题则 CA 证书已经如何进行认证的，涉及到的对称与非对称加密要去了解。

如果你做完上上面这些，那么拿到一个心仪的 offer 是没有丝毫问题的。接下来该写简历了。

## 简历怎么写
我没有办法教你如何从头到尾写一份简历，但是我可以分享我的几个原则：
1. 一份简历里最多只有两种字体、最多只有两种字号，强调的地方可以加粗
2. 基础信息只写重要的
3. 邮箱和联系方式要醒目
4. 项目经验把最重要最熟悉的写第一个，不要按照时间顺序排，面试官90%的时候都是问第一个
5. 项目经验不要记流水账，写重要贡献和关键技术，用数据量化。比如系统优化，给性能提升了30%，比如使用了多线程等
6. 会的技能写上去，会一点但是一问就卡壳的技能不要写上去，并不是写的多就好，而是如果面试官针对你的一个技能点不断挖掘，如果你始终能够抗住，那么就可以吃鸡了。

## 学习
后端技术那么多，mysql、redis、kafka、rocketmq、rpc、docker、k8s、servicemesh、etcd、zookeeper 我该先学什么？原则很简单，选择你现在正在用的优先学。因此学习了mysql、redis、kafka，在这阶段的学习主要通过一些专栏文章、博客和GitHub来完成学习，mysql 推荐极客时间的《MySQL 45讲》，redis 推荐《Redis 的设计与实现》，kafka 看了核心设计。工作一年以后，在原来京东的岗位上我觉得无法成长了，可能是因为不在核心的部门，一年我觉得就遇到瓶颈了。于是换工作来到了字节跳动。在这里有接触了很多新的东西。接下来我继续学习什么？
1. devops 是一种软件工程的开发与交付理念，致力于管理整个软件的开发周期，将整个开发周期整合到一套流程中。
2. go 语言的开发效率比c++要高效得多，而且 go 与云结合的非常好，未来前景不错。
3. 分布式 KV 存储，现在以 etcd、tikv 等一批分布式 kv 存储在大型分布式系统中扮演越来越重要的角色，那么选择一款分布式存储系统进行学习，可以大大提升自己对分布式服务的理解。
4. 服务治理，微服务架构目前已经是最主流的架构了，复杂业务场景下微服务多达几百上千个。公司业务线繁多的情况下，微服务更是数不胜数，那么如何治理这些服务成为了一大难题，幸好有 service mesh 的出现为服务治理带来了很大的便捷。当然服务的治理还有更广泛的东西，包括日志收集、监控报警等。
5. 阿里、腾讯的云服务收入正逐年大幅上涨，说明现在越来越多的企业将服务迁移到了云上，那么掌握云计算相关的技术也是一名开发必备的技术。云目前主要都是依托于 docker，服务的编排则是基于 k8s。

>准备

>## 算法题
>事先已经看过别人的社招面经知道头条每轮技术面都有算法题，而这一块平时练习的比较少，校招时刷的题也忘记了很多。因此系统复习的时候算法题还是花了比较多时间的。先是快速刷完了剑指 offer，这个校招时已经刷过两边了，因此现在刷起来会相对快一些。然后就是啃 LeetCode 的题了，LeetCode 的题比较多，想在短短几周内刷完基本是不可能的，因此我主要按照类型去刷，每个类型刷几道就会比较有感觉了。比如链表的题优先考虑递归和双指针来解决，栈和队列的题优先考虑用两个栈或队列来解决，树的题基本都是递归等。不过数组和字符串的题一般比较灵活，这种题只能尽量多刷了。平时要上班刷题也不方便，我采用的方法就是看题，用手机打开 LeetCode 的网站，看完题目后直接想解决方案，脑子里大概捋一下代码怎么写，能想到的就过，想不出的就看看别人的解法，用这个方法刷起来就很快。用这种方法你可能会担心面试时题写不完整，其实不用太担心，因为面试的时候面试官看你写的核心思路是正确的，边界处理是对的基本就过了，面试时间比较有限。

>## 理论基础
>基础这一块主要以快速复习为主，主要是语言（我主要用C++，所以复习C++）、操作系统和网络编程。校招这一块会问题的比较多，社招这一块问的比较少，但是如果这一块打不上来就比较尴尬了。语言就不说了，这一块大家应该都知道会考些什么，校招的时候毕竟都疯狂准备过。操作系统就看内存管理、进程管理和文件系统，一般虚拟内存问的多。网络编程这块就包括 TCP/IP 协议，HTTP协议，网络安全三个方面。TCP/IP主要就是三次握手，四次挥手，TIME_WAIT 的作用等这些常考的题了。HTTP 协议考察 HTTP 协议的返回码、HTTP 的方法等。需要特别指出的是 HTTPS 加密的详细过程要非常透彻，不然容易产生一种感觉好像都清楚了，但是一问就有点说不清楚。最后就是网络安全，主要考察也是 WEB 安全，包括XSS，CSRF，SQL注入等。

>后端技术
>这里的后端技术主要指工作中要用到的一些基础组件，一些常见的后端架构设计。主要准备了MySQL、Redis、消息队列、zookeeper、分布式系统架构设计和docker。MySQL 主要看了极客时间的 《MySQL 45讲》，关于事务、索引、锁以及 binlog 和 redolog 都讲的非常好，也是面试最爱考的，除此之外对数据库的读写分离、分库分表也要掌握。没有任何利益相关，决不是打广告。Redis 主要看了《Redis 的设计与实现》，然后自己再总结了一下 Redis 的使用场景，以及 Redis 实现分布式锁基本 Redis 就没有问题了。消息队列的开源软件比较多，我主要选择 Kafka 来学习，主要看官网文档，极客时间的《Kafka 核心技术与实战》，和一些技术文章等。不过极客时间的《Kafka 核心技术与实战》，我觉得讲的比较一般，不是很建议。分布式系统的就准备CAP理论、BASE理论、限流、熔断、一致性***算法、主从架构、集群架构、异地多活、负载均衡、分层架构、微服务等。

>深挖项目
>没有参与开源项目的经验，工作中做的项目也很一般，项目这块我实在没什么太多拿的出手的，不过还是要挖掘一下，毕竟这一块是逃不掉。我说几个我思考的点吧：

>找项目中相对而言具有亮点的地方。比如我用 redis 实现了一个延时队列，然后对这个延时队列我通过分片来解决瓶颈，通过分发来加快处理速度。可以看我的《基于REDIS实现延时任务》一文。这个虽然不是什么复杂技术，但是可以将其考虑全面可以展示出自己具有一定的架构能力。
>找项目中复杂的地方。如果你做的项目中有复杂的地方，即使不是你做的，也可以拿来说，前提是你要搞得非常清楚来。
>量化指标。一个接口原来有性能问题，比如你做了一个小的优化，将其 TP99 的耗时从原来的 500ms 优化至多少 200ms。
>赋能整个团队。在开发业务的过程中肯定会遇到一些重复的工作，或者可以复用的服务。你可以开发了某个工具或者服务化了某个功能推广到了全组使用，给公司创造了价值。
